## Context

当前插件的中英字幕都渲染在同一个 overlay 根节点下，并由内容脚本按时间持续刷新文本。现有布局依赖固定的底部居中定位，适合默认观看，但在图表课程场景下很容易挡住画面关键信息。由于字幕本身已经是一个独立容器，最自然的做法是在不改变字幕渲染管线的前提下，为该容器增加拖动偏移。

本次目标是“把中英字幕作为一个整体拖动”。这意味着交互层只移动字幕容器的位置，不拆分主字幕和副字幕，不触碰字幕内容来源、配音同步或 provider 逻辑。

## Goals / Non-Goals

**Goals:**
- 允许用户通过鼠标拖动字幕覆盖层。
- 保持中英字幕始终作为同一容器整体移动。
- 让拖动实现与现有字幕刷新、全屏挂载和中文配音播放逻辑解耦。
- 避免拖动与插件面板按钮、选择框等交互冲突。

**Non-Goals:**
- 不支持分别拖动中英文字幕。
- 不在本次引入键盘微调、触屏拖拽或位置预设系统。
- 不改动字幕解析、字幕同步或中文配音的底层逻辑。

## Decisions

### 1. 以字幕 overlay 容器作为唯一拖动对象
- 决策：拖动能力挂在当前字幕显示容器上，中英文两行跟随同一个容器位移。
- 原因：这与“整体移动”需求直接对应，也能避免两行字幕位置失配。
- 备选方案：分别给主/副字幕绑定拖动；会破坏双语对齐，不符合本次目标。

### 2. 用运行时偏移量覆盖默认底部居中布局
- 决策：保留默认底部居中样式作为初始位置，在用户拖动后用运行时偏移量更新容器位置。
- 原因：这样既保留当前默认体验，也能以最小改动支持拖动。
- 备选方案：完全改用绝对坐标布局；会让默认样式和后续全屏挂载逻辑更复杂。

### 3. 仅在字幕容器命中时开始拖动
- 决策：只有按下字幕容器本身时才进入拖动态，插件面板和其他 UI 不参与拖动。
- 原因：减少与面板按钮、输入框、选择器等控件的冲突。
- 备选方案：整个 overlay 根节点都可拖动；误触概率更高。

### 4. 拖动过程保持字幕刷新独立
- 决策：拖动只更新位置状态，不暂停字幕刷新，也不改动当前 cue 的渲染逻辑。
- 原因：位置和内容是两个独立维度，拆开处理更稳定。
- 备选方案：拖动时暂停渲染；没有必要，且容易引入闪烁。

## Risks / Trade-offs

- [拖动时选中文本或触发页面默认拖拽] → 在拖动期间阻止默认行为，并为字幕容器设置合适的鼠标样式与选择策略。
- [全屏切换后位置计算异常] → 让拖动偏移绑定到当前 overlay 容器，并验证普通播放与全屏挂载切换。
- [字幕被拖出可视区域] → 在实现中限制拖动结果，确保容器至少保留可见区域。

## Migration Plan

1. 为字幕显示容器增加可拖动样式和事件绑定入口。
2. 增加拖动状态与偏移量管理，并将位置更新应用到整体字幕容器。
3. 限制拖动边界，避免字幕完全移出可视区域。
4. 验证普通播放、全屏、双语显示、插件面板交互等场景。
5. 如需回滚，只需移除拖动事件与位置状态逻辑，不涉及数据迁移。
