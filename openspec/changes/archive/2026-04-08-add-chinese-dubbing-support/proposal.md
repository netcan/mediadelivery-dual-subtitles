## Why

当前扩展只能在 MediaDelivery 播放器中叠加双语字幕，无法为英文原声视频提供可切换的中文配音播放体验。对于已有中文字幕的课程视频，先基于字幕驱动生成中文配音，并在播放时同步切换音频，比直接引入实时 ASR/翻译链路更容易形成可用闭环，也为后续接入本地模型、自定义 API 和 ASR 增强能力预留了架构空间。

## What Changes

- 新增“中文配音”能力：在英文原声 + 中文字幕场景下，支持发起中文配音生成任务，并在播放时切换到生成后的中文配音音频。
- 新增 Provider 配置能力：允许用户配置云端 API、自定义 HTTP API、`localhost` 本地服务或 OpenAI-compatible 接口，用于接入翻译改写和 TTS/配音模型。
- 新增任务状态与结果接入能力：扩展可创建配音任务、轮询结果，并加载返回的中文配音音频与相关元数据。
- 新增播放同步控制：在启用中文配音时，扩展负责协调视频与外挂中文音频的播放、暂停、跳转和倍速同步，并保留中文字幕显示。
- 明确首期范围：ASR 作为后续增强能力预留，不纳入本次实现范围。

## Capabilities

### New Capabilities
- `chinese-dubbing-playback`: 基于现有中文字幕发起中文配音生成，并在播放器中切换和同步播放中文配音音轨。
- `model-provider-config`: 配置并管理用于配音流水线的模型服务提供方，支持自定义 `baseURL`、`apiKey` 和模型名称。

### Modified Capabilities
- 无

## Impact

- 影响前端扩展注入逻辑、播放控制逻辑、设置存储和面板 UI。
- 需要定义与外部配音服务交互的任务接口、结果元数据格式和错误处理策略。
- 可能需要补充扩展权限与网络访问策略，以支持用户配置的远程或本地服务地址。
- 后续可在不推翻当前接口的前提下接入 ASR、说话人分离和更细粒度的时间对齐能力。
