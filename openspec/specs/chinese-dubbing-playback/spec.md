## Purpose

定义插件如何基于现有中文字幕创建、跟踪和播放中文配音结果，以及在启用中文配音期间保持字幕与音频体验一致。
## Requirements
### Requirement: 用户可以基于现有中文字幕发起中文配音任务
系统 MUST 在检测到可用中文字幕时，允许用户从扩展面板发起中文配音生成任务，并将字幕内容、时间轴和所选 Provider 配置提交给外部服务。

#### Scenario: 可用中文字幕时显示任务入口
- **WHEN** 扩展检测到当前视频存在可读取的中文字幕轨或用户已导入中文字幕文件
- **THEN** 扩展显示可用的“生成中文配音”任务入口

#### Scenario: 缺少中文字幕时阻止任务发起
- **WHEN** 当前视频不存在可读取的中文字幕，且用户未导入任何中文字幕文件
- **THEN** 扩展 MUST 阻止发起中文配音任务，并向用户提示当前前置条件不满足

### Requirement: 系统可以查询并展示中文配音任务状态
系统 MUST 支持跟踪中文配音任务的状态，并在排队、处理中、部分可播、成功和失败等阶段向用户展示明确状态；当 Provider 已生成连续可播放分段时，系统 MUST 能识别任务已进入可提前播放状态。

#### Scenario: 任务处理中
- **WHEN** 用户成功发起中文配音任务且外部服务尚未返回任何可播放分段
- **THEN** 扩展显示任务进行中状态，并允许后续轮询或刷新结果

#### Scenario: 部分分段已可播放
- **WHEN** 外部服务返回任务仍在运行，但已提供连续可播放的中文配音分段与可播放进度
- **THEN** 扩展 MUST 将该任务标记为“可提前播放”或等价状态，并继续轮询后续进度

#### Scenario: 任务失败
- **WHEN** 外部服务返回任务失败、鉴权失败或模型处理错误
- **THEN** 扩展 MUST 向用户展示可理解的失败状态和错误信息

### Requirement: 用户可以切换到生成后的中文配音播放
系统 MUST 在中文配音任务成功后允许用户启用中文配音，并在播放时将原视频与中文配音音频保持同步；对于分段播放和最终整段播放，系统 MUST 优先支持通过扩展代理生成的 `blob:` 或等价本地媒体引用进行播放，而不是依赖页面直接加载远端 Provider 音频 URL。

#### Scenario: 使用代理音频源启用中文配音
- **WHEN** 当前视频存在可用的中文配音结果且用户启用中文配音
- **THEN** 扩展 MUST 先通过扩展后台获取可播放的本地媒体引用，再将其绑定到页面音频播放对象

#### Scenario: 分段切换时复用代理链路
- **WHEN** 中文配音播放从一个已完成分段切换到下一个已完成分段
- **THEN** 扩展 MUST 继续使用扩展代理获取的本地媒体引用播放下一段，而不是直接切换到远端 Provider URL

#### Scenario: 切回原声
- **WHEN** 用户关闭中文配音模式
- **THEN** 扩展 MUST 停止外挂中文配音播放并恢复原始视频的默认音频行为

### Requirement: 启用中文配音时仍须保留中文字幕显示
系统 MUST 在中文配音启用期间继续支持中文字幕显示，且不因切换到中文配音而丢失字幕覆盖层。

#### Scenario: 中文配音与中文字幕同时启用
- **WHEN** 用户启用中文配音且当前存在可用中文字幕
- **THEN** 扩展 MUST 继续显示中文字幕，除非用户显式关闭字幕显示

#### Scenario: 中文配音结果返回附带新字幕
- **WHEN** 中文配音任务结果同时返回新的字幕资源
- **THEN** 扩展 MUST 至少保留一种可用中文字幕显示路径，不得出现启用中文配音后无中文字幕可见的状态

### Requirement: 中文配音播放必须控制媒体缓存窗口
系统 MUST 在分段中文配音播放过程中控制媒体缓存窗口，避免因为持续保留全部音频分段而导致内存无限增长。

#### Scenario: 保留有限播放窗口
- **WHEN** 用户正在播放某个分段，且系统已预取相邻分段
- **THEN** 扩展 MUST 仅保留有限数量的当前/相邻分段媒体引用，而不是缓存整条视频的全部音频分段

#### Scenario: 跳转后丢弃旧窗口
- **WHEN** 用户跳转到一个远离当前播放位置的新时间点
- **THEN** 扩展 MUST 释放旧播放窗口中不再需要的媒体引用，并围绕新的目标时间重建缓存窗口
